

# 📰 Newspaper Revenue Generation: A Clustering Approach

> **An advanced Machine Learning pipeline leveraging Google Analytics data to uncover online revenue generation patterns for Bangladesh's leading daily newspapers.**

## 📌 Project Overview
The transition from print to digital media has fundamentally shifted how newspapers generate revenue. This project explores the online monetization models of Bangladesh's top two daily newspapers: **The Daily Star** (English) and **Prothom Alo** (Bengali). 

By applying nine distinct unsupervised machine learning algorithms to website traffic data, this research uncovers hidden patterns in user engagement, session duration, and page value. The resulting models segment web pages into distinct profitability clusters, empowering publishers to make data-driven decisions regarding content placement, ad targeting, and reader retention.

---

## 📊 The Dataset
The dataset comprises organic Google Analytics data collected over a specific period in 2022. It contains 500 high-density samples segmented by specific news portals.

### Feature Dictionary
| Feature | Data Type | Description |
| :--- | :--- | :--- |
| **Page** | `String` | The specific URL or title of the visited news article/portal. *(Dropped during training to prevent string-based noise).* |
| **Pageviews** | `Numeric` | Total number of pages viewed, including repeated views by the same user. |
| **Unique Pageviews**| `Numeric` | Number of individual sessions during which a specified page was viewed at least once. |
| **Avg. Time on Page**| `Numeric` | The average duration (in seconds/minutes) users spent viewing a specific page. |
| **Entrances** | `Numeric` | The number of times visitors entered the website through this specific page. |
| **Bounce Rate** | `Percentage` | Percentage of single-page sessions with zero subsequent interactions. |
| **% Exit** | `Percentage` | The frequency at which users exit the domain from a specific page. |
| **Page Value** | `Numeric` | **(Target Variable equivalent)** The average financial value generated by the page via transactions or goal completions. |

## ⚙️ Machine Learning Pipeline

This project utilizes a highly optimized data science pipeline, built primarily using `Pandas` for manipulation and `PyCaret` for low-code, high-efficiency machine learning experimentation.

### 1. Data Engineering & Preprocessing
Google Analytics exports are notoriously noisy. The preprocessing phase involved:
* **Regex Filtering:** Utilizing Python's `re` and Pandas' string manipulation to strip out anomalous symbols and successfully parse both English and Bengali characters.
* **Missing Values:** Numerical and categorical imputations were executed to handle null data points seamlessly.
* **Feature Pruning:** Text-heavy arrays (like Page URLs) were dropped to isolate the behavioral and financial metrics for pure mathematical clustering.

### 2. Normalization & Dimensionality Reduction
* **Standard Scaler (Z-Score):** Applied to standardize the vastly different numerical scales (e.g., percentages vs. millions of pageviews).
* **Principal Component Analysis (PCA):** Implemented with `n_components=4` to reduce noise, eliminate multicollinearity, and extract the most optimal hyperparameters for the clustering phase.

### 3. Clustering Architecture
The study benchmarked nine distinct algorithms to find the perfect fit for media revenue data:
* *Spectral Clustering, K-Means, DBSCAN, Agglomerative Hierarchical, Birch, Affinity Propagation, Mean Shift, Gaussian Mixture, and Optics.*

## 🏆 Key Findings & Experimental Results

Model performance was rigorously evaluated using the **Silhouette Score**, measuring how similar an object is to its own cluster compared to other clusters.

### Top Performing Model: Spectral Clustering
Spectral Clustering outperformed all traditional models (including K-Means and DBSCAN) by successfully handling the non-linear dimensional structure of web traffic behavior.
* **The Daily Star (English):** Achieved an **89% Silhouette Score**.
* **Prothom Alo (Bengali):** Achieved an **83% Silhouette Score**.

### Business Insights & Cluster Breakdown
Using `Plotly Express` Treemaps, the Spectral Clustering model successfully mapped the websites into financial tiers:

#### **The Daily Star (4 Distinct Clusters)**
1. **Most Profitable:** Users spent >1 minute per page, generating \$0.10 to \$1.00 per unique visitor. 
2. **Profitable:** Averaged \$0.01 per unique visitor but with massive traffic volumes.
3. **Average Profitable:** Moderate traffic, moderate bounce rate.
4. **Least Profitable:** Low retention pages driving minimal ad-revenue.

#### **Prothom Alo (5 Distinct Clusters)**
1. **Cluster 0 (Apex):** Generated the vast majority of overall revenue due to immense scaling and traffic volume.
2. **Clusters 1, 2, 3:** Represent closely related sub-segments. Cluster 1 holds a moderate revenue stream, while 2 and 3 represent high-bounce, low-revenue pages.
3. **Cluster 4:** A niche category of pages that, despite lower traffic, generate a disproportionately high amount of targeted revenue.

## 💻 Getting Started

### Prerequisites
To run the Jupyter Notebook, ensure you have Python 3.8+ installed along with the following primary libraries:
```bash
pip install pandas numpy pycaret plotly scikit-learn
```

### Usage
1. Clone the repository:
   ```bash
   git clone https://github.com/evanmahmud/Newspaper-Revenue-Model.git
   ```
2. Navigate to the directory and launch Jupyter:
   ```bash
   cd Newspaper-Revenue-Model
   jupyter notebook
   ```
3. Open and run **`Newspaper.ipynb`** to view the interactive Plotly graphs and PyCaret training pipelines.
